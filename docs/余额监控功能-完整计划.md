## 一、背景与目标

当前 Token Cost App 通过读取本地 SQLite / JSONL 数据计算 token 用量，订阅计划完全依赖用户手动在设置面板选择预设档位。无法获取实时余额/credit 信息。

参考 opencode-bar（`opgginc/opencode-bar`）的架构，本方案将为 app 增加：
1. 从 `auth.json` / `~/.codex/auth.json` 安全读取 API key
2. 调用各 provider 官方 API 获取实时余额
3. 在仪表盘、各页面、菜单栏展示余额信息
4. 基于实时数据自动匹配订阅计划预设

## 二、opencode-bar 余额实现参考

opencode-bar 的核心架构：

| 组件 | 职责 |
|------|------|
| `TokenManager` 单例 | 从 `~/.local/share/opencode/auth.json` 读取所有 provider 的 key/token |
| `ProviderProtocol` | `func fetch() async throws -> ProviderResult` |
| `ProviderResult` | 封装 `ProviderUsage` (payAsYouGo/quotaBased) + `DetailedUsage` |
| 每个 Provider | 实现 `fetch()`，调用各自的 API 端点 |

关键 API 端点：

| Provider | 端点 | 返回 |
|----------|------|------|
| OpenRouter | `GET /api/v1/credits` + `GET /api/v1/key` | total_credits, used, monthly_cost |
| OpenCode Go | `GET https://api.opencode.ai/v1/credits` | total_credits, used_credits, remaining_credits |
| OpenCode Zen | 本地 CLI `opencode stats --days 7 --models` | Total Cost, MODEL USAGE 表格 |
| Codex | `GET https://chatgpt.com/backend-api/wham/usage` | rate_limit windows, plan_type, credits |
| MiniMax | `GET .../coding_plan/remains` | 5h/周 使用次数 |

## 四、安全审查

| 维度 | 风险 | 缓解 |
|------|------|------|
| API Key 暴露 | auth.json 内容在内存/日志中泄漏 | 内存缓存 30s 后清空；`CustomStringConvertible` 重写为 `***`；不写入磁盘 |
| 网络请求 | 首次在 app 中引入网络通信 | `URLSession` ephemeral 配置；HTTPS only；不经过第三方服务器 |
| auth.json 读取 | SafeFileStore 限定了读写 root | auth.json 不在该 root 内，需独立只读访问 `Data(contentsOf:)` |
| 沙箱限制 | 当前无 entitlements 文件 | 读取 `~/.local/share/opencode/auth.json` 不受限 |
| 频率限制 | 频繁 API 调用 | 默认 5 分钟间隔；失败后 exponential backoff |
| OAuth 过期 | Codex token 刷新失败 | 静默跳过；UI 显示"需要重新登录" |
| CLI 进程安全 | Zen 余额通过 Process 执行 opencode | 参数写死，不拼接用户输入；stdout 解析后丢弃 |
| 用户 opt-in | 默认关闭余额监控 | 不影响 app 原有"纯本地"承诺 |

## 五、余额使用预估梯度

```
0%     → 🟢「未使用」   (灰色, dimmed)
1-49%  → 🟢「剩余充足」 (green)
50-79% → 🟡「适中」     (yellow)
80-94% → 🟠「接近上限」 (orange)
95-99% → 🔴「即将用尽」 (red)
100%+  → 🔴「已超额」   (red, bold)
```

对于 OpenCode Zen（按量计费），不显示百分比，而用「本月累计 $X.XX」（无上限）。

## 九、关键数据模型

```swift
enum BalanceProviderKind: String, Codable, CaseIterable {
    case opencodeGo
    case opencodeZen
    case codexCodingPlan

struct BalanceSnapshot: Codable, Sendable {
    let provider: BalanceProviderKind

    let primaryWindowResetAt: Date?

    // Zen 专用
    let totalCostUSD: Double?
    let avgCostPerDayUSD: Double?
}
```

## 十、OpenCode Zen CLI 方案确认

Zen 余额通过内嵌 `Process` 执行本地 `opencode` CLI 获取：

- 命令：`opencode stats --days 7 --models`
- 二进制发现：PATH → login shell PATH → 硬编码回退
- 超时：60s
- 安全：参数写死，stdout 解析后丢弃
- 回退：CLI 不可用时标记 unavailable，不阻塞其他 provider
- 全部在 app 内自动完成，无需用户操作
