# 功能模块关联清单

> 本文档记录所有已实现的功能、对应模块文件、数据管线与交叉依赖。  
> 修改或删除任何功能前，**必须先阅读本文档**确认影响范围——避免 agent 把局部功能修改误解为整个 app 的替换。

## 1. 数据源管线

### 1.1 OpenCode 管线（SQLite → 仪表盘）

| 阶段 | 文件 | 职责 |
|------|------|------|
| 来源发现 | `SourceDiscoveryService.swift` | 扫描文件系统发现 SQLite 数据库 |
| 数据读取 | `TokenDatabaseClient.swift` | `json_extract` 查询 → `DashboardPayload` |
| 数据模型 | `Models.swift` | `DashboardPayload`, `RawRow`, `Summary` |
| 分析引擎 | `DashboardAnalytics.swift` | Overview/Cache/Provider Rank/Model Comparison/Trend |
| 数据 Store | `TokenCostModel.swift` | MVVM 状态管理、快照持久化、设置加载 |
| 展示页面 | `DetailView.swift`, `SidebarView.swift`, `OpenCodePageView.swift` | 侧边栏 + 详情仪表盘 |

**关键约束**：OpenCode SQLite 中 `$.tokens.input` 字段**已是非缓存值**。代码中所有 `actualTokens` 计算均用 `input + output + reasoning`，不再额外减去 `cacheRead` 或 `cacheWrite`。

### 1.2 Codex 管线（JSONL → 仪表盘）

| 阶段 | 文件 | 职责 |
|------|------|------|
| 来源配置 | `SourceProfiles.swift` | Codex 默认扫描路径 |
| 子进程调用 | `CodexHelperRunner.swift` → `CodexTokenCostHelper` | 外部进程采集 session 文件 |
| 数据解析 | `CodexSessionCollector.swift` | JSONL 逐行解析 → `CodexDashboardPayload` |
| 数据模型 | `CodexModels.swift` | `CodexTokenUsage`, `CodexSessionSummary` |
| 分析引擎 | `CodexAnalytics.swift` | 趋势点聚合、session 排序 |
| 数据 Store | `CodexSessionModel.swift` | MVVM 状态管理、快照持久化 |
| 展示页面 | `CodexPageView.swift` | Session 列表 + 趋势图 |

**数据差异**：Codex JSONL 中 `inputTokens` 包含缓存，因此 `actualInputTokens = max(inputTokens - cachedInputTokens, 0)`。与 OpenCode 管线不同，不做 `cacheWrite` 减法（Codex 无此字段）。

## 2. 完整功能清单

### 2.1 Token 用量统计（两个数据源各自独立）

- OpenCode: 每日趋势、实际/缓存 token、消息数、活跃天数
- Codex: 每日趋势、session 级 input/output/reasoning/cached/actual 明细
- **关键文件**: `DashboardAnalytics.swift`, `CodexAnalytics.swift`, `DetailView.swift`, `CodexPageView.swift`
- **注意**: OpenCode 和 Codex 的 actual token 公式本质不同——OpenCode 的 input 已是非缓存，Codex 的 input 含缓存需要减法。不能简单套用同一公式。

### 2.2 费用分析（API + 订阅双口径）

- 每模型 API 定价表（`TokenCostPricingCatalog.zenPricing`）
- Provider 订阅成本覆盖：订阅 > 按量 API > raw db cost
- Provider 性价比排行（tokens/$）
- 模型价格对比（成本分摊）
- **关键文件**: `DashboardAnalytics.swift` (lines 736-818), `BillingPlanCatalog.swift`, `CodexBilling.swift`
- **展示**: `TotalView.swift`（总览页切换 API/订阅口径）, `DetailView.swift`（Provider 排行）

### 2.3 Provider 订阅计划管理

- 四提供商的官方订阅 / Token Plan 预设：OpenCode Go/Zen、ChatGPT Plus/Pro/Business、MiniMax Token Plan、Xiaomi MiMo Token Plan
- 自定义 USD 月费：用于应对官方价格变更
- **关键文件**: `BillingPlanCatalog.swift`, `AppPreferences.swift`, `AppPreferencesModel.swift`
- **UI**: `SettingsView.swift`（计费方案卡片）, `TotalView.swift`（总览使用 resolved plan）
- **分析注入**: `TokenCostDashboardAnalytics` 接收 `billingOverridesByProviderKey`，应用于 provider effective cost

### 2.4 中英双语本地化

- 所有 UI 文本通过 `AppLocalization.text()` / `AppLocalization.format()` 获取
- 语言设置持久化在 `AppPreferences.language`
- **关键文件**: `Localization.swift`, `Resources/*.lproj/Localizable.strings`
- **依赖链**: `AppLocalization.currentLanguage` → `.lproj` bundle 读取 → 全 UI 刷新
- **约束**: 技术术语保留英文（token、API、OpenCode、Codex、ChatGPT Plus 等）

### 2.5 多主题支持

- 4 种主题色：海湾蓝、森林绿、暮光橙、极光紫
- **关键文件**: `Theme.swift`, `ThemePalette.swift`
- **依赖项**: 所有 View 通过 `palette: TokenCostPalette` 引用颜色，不硬编码

### 2.6 设置面板

- OpenCode: 扫描目录/数据库文件、自动扫描、显示零用量 provider
- Codex: session 目录/文件、自动刷新
- 全局: 语言、总览计价模式、计费方案管理
- **关键文件**: `SettingsView.swift`, `SettingsStore.swift`, `SafeFileStore.swift`

### 2.7 内置计费文档查看

- 从 app bundle 加载 `Pricing.md`，用系统 `AttributedString(markdown:)` 渲染
- 不网络、不任意路径
- **关键文件**: `PricingDocView.swift`, `Resources/Pricing.md`

### 2.8 安全边界

- 所有源数据只读（OpenCode SQLite / Codex JSONL）
- 配置/快照写入 `~/Library/Application Support/`，均经 `SafeFileStore` 路径验证
- 纯本地，零网络
- **关键文件**: `SafeFileStore.swift`, `SECURITY.md`

### 2.9 双源总计页

- 融合 OpenCode + Codex 的实际输入 token 和费用
- 支持 OpenCode 计价模式切换（API vs 订阅）
- **关键文件**: `TotalView.swift`

### 2.10 菜单栏

- 刷新 OpenCode / Codex、打开主窗口、打开设置
- **关键文件**: `TokenCostCommands.swift`, `MenuBarView.swift`

## 4. 约束与边界（修改功能前必读）

1. **App 纯本地运行**——不移除、不接入网络。所有数据来自本地文件系统。
2. **源数据只读**——不修改 OpenCode SQLite 或 Codex JSONL 文件。只写配置到 Application Support。
3. **OpenCode vs Codex 公式差异**——不要强行统一两端的 actual token 公式。两端数据模型不同。
4. **input 字段语义**——OpenCode SQLite 的 `input` 已是非缓存值。不要再做 `input - cacheRead - cacheWrite` 减法。
5. **订阅费用回退机制**——`BillingPlanCatalog` 解析失败时会 fallback 到 `TokenCostPricingCatalog.subscriptionMonthlyCosts`。不要移除后者。
6. **不要引入第三方依赖**——当前仅有 sqlite3 系统库。不要加 Markdown 渲染库或网络库。
7. **所有 UI 文案走 `AppLocalization`**——不要在任何 View 中硬编码中/英文字符串。
8. **代码注释只在必要时添加**——公开 API 需要 doc comment 说明参数/行为；内部逻辑优先通过清晰的命名自我表达。
