# 开发手册

## 1. 项目概述

**Codex Token Cost App**（全称 Token Cost App - OC Codex Ver）是一个 macOS 原生桌面应用，用于可视化 AI 编程工具（OpenCode / Codex）产生的 token 用量和费用数据。

- **语言**: Swift 6.0+
- **UI 框架**: SwiftUI + AppKit (部分)
- **构建系统**: Swift Package Manager (SPM)
- **最低系统**: macOS 14.0 (Sonoma)
- **仓库**: [GitHub](#) · MIT License

### 数据源

| 数据源 | 格式 | 默认路径 |
|--------|------|---------|
| **OpenCode** | SQLite 数据库 | `~/.local/share/opencode/opencode.db` |
| **Codex** | JSONL 文本文件 | `~/.codex/sessions/` |

## 3. 模块架构

### 3.1 CodexTokenCostCore（核心库）

最底层模块，不依赖 AppKit / SwiftUI。被 App 和 Helper 共享引用。

| 文件 | 职责 |
|------|------|
| `Models.swift` | 数据模型：`TokenCostSource`, `TokenCostSettings`, `DashboardPayload`, `UsageAggregateRow` |
| `CodexModels.swift` | Codex 专用模型：`CodexTokenUsage`, `CodexSessionSummary`, `CodexDashboardPayload` |
| `DashboardAnalytics.swift` | 分析引擎：将原始行转为 Overview / Cache / Provider Rank / Model Comparison / Distribution / Trend / Stacked。**含模型定价目录**（`TokenCostPricingCatalog`） |
| `CodexAnalytics.swift` | Codex 分析：session 排序、日期桶聚合、趋势点生成 |
| `CodexBilling.swift` | 计费常量 (GPT Plus $20/月) |
| `TokenDatabaseClient.swift` | SQLite 客户端：探针 + `json_extract` 查询 + 构建 `DashboardPayload` |
| `SourceDiscoveryService.swift` | 文件系统扫描：遍历目录发现数据库文件 |
| `CodexSessionCollector.swift` | JSONL session 解析器：读取每行 JSON、提取 `session_meta` 和 `event_msg(token_count)` |
| `SourceProfiles.swift` | OpenCode / Codex 的默认路径、文件模式、扫描深度等配置 |
| `SettingsStore.swift` | 设置 + 快照文件的读写和轮转 |
| `SafeFileStore.swift` | 沙箱内安全文件操作（防止路径穿越） |
| `PathUtilities.swift` | 路径标准化、descent 检查 |
| `AppPaths.swift` / `CodexSessionPaths.swift` | 路径常量 |
| `Theme.swift` | 主题枚举 |

### 3.2 CodexTokenCostApp（主应用）

依赖 Core 模块。`@main` 入口。

| 目录 | 职责 |
|------|------|
| `App/` | `@main` 入口、AppDelegate、路径常量、菜单命令 |
| `Stores/` | `TokenCostModel`（OpenCode 数据层）、`CodexSessionModel`（Codex 数据层）、`CodexSnapshotStore` |
| `Views/` | 所有 SwiftUI 视图（6 个主要页面 + 通用组件） |
| `Services/` | `CodexHelperRunner`（通过 `Process` 调起 Helper 子进程） |
| `Support/` | `ThemePalette`（主题调色板）、`SeriesPalette`（图表配色） |

### 3.3 CodexTokenCostHelper（辅助进程）

独立可执行文件，由主 App 通过子进程方式调用。

| 文件 | 职责 |
|------|------|
| `main.swift` | CLI 入口：解析 `--source-root` / `--manual-source-path` 参数 |
| `CodexSessionCollector.swift` | 采集 Codex session 文件，解析为 `CodexDashboardPayload`，输出 JSON 到 stdout |

## 4. 数据流

### 4.1 OpenCode 数据管线

```
[文件系统扫描]
SourceDiscoveryService
    ↓ discover()
[SQLite 数据库列表] → TokenCostSource[]
    ↓ 用户选择来源
[TokenDatabaseClient.loadPayload()]
    ↓ SELECT json_extract(...) FROM message
[UsageAggregateRow[]]
    ↓ buildPayload()
[DashboardPayload]
    ↓ TokenCostDashboardAnalytics 初始化
[Overview / Cache / Provider / Model / Distribution / Trend]
    ↓ 各 View 读取
[UI 渲染]
```

**关键 SQL**: 使用 `json_extract(data, '$.tokens.input')` 等从 JSON 列提取字段，按 `date, model_id, provider_id` 分组聚合。

### 4.2 Codex 数据管线

```
[用户在设置中配置 session 目录/文件]
    ↓
[CodexHelperRunner.loadPayload()]
    ↓ 启动子进程
[CodexTokenCostHelper (外部进程)]
    ↓ 遍历 JSONL 文件
[CodexSessionCollector.loadPayload()]
    ↓ 逐行解析 session_meta / event_msg
[CodexDashboardPayload]
    ↓ stdout JSON → 主进程解码
[CodexSessionModel.payload]
    ↓ CodexDashboardAnalytics 分析
[UI 渲染: 趋势图 + Session 列表]
```

**Error 处理**: Helper 进程超时 15s，失败自动回退到本地快照 `dist/snapshots/codex/`。

## 5. 构建与运行

### 5.1 本地开发

```bash
# 进入项目根目录
cd Codex-Token-Cost-App

# 编译（Debug）
swift build

# 编译并运行（使用脚本）
bash script/build_and_run_codex.sh run

# 仅构建 .app（不运行）
bash script/build_and_run_codex.sh build

# LLDB 调试
bash script/build_and_run_codex.sh debug

# 查看日志流
bash script/build_and_run_codex.sh logs
```

### 5.2 Release 构建

```bash
swift build -c release
bash script/build_and_run_codex.sh build
# 产物位于 dist/releases/<timestamp>/
```

### 5.3 依赖

- macOS 14.0+
- Xcode / Command Line Tools (提供 `xcrun`, `swift`, `lldb`)
- SQLite3 (系统自带)
- 无其他外部依赖

## 6. 版本管理

### 语义化版本 (SemVer)

```
MAJOR.MINOR.PATCH
0.2.0
```

| 变更类型 | 版本 | Git 操作 |
|---------|------|---------|
| 大重构 / 不兼容变更 | MAJOR | `git tag -a v1.0.0 -m "..."` |
| 新功能 | MINOR | `git tag -a v0.3.0 -m "..."` |
| Bug 修复 | PATCH | `git tag -a v0.2.1 -m "..."` |

### 发布流程

```
1. 更新 CHANGELOG.md（写入本次变更）
2. 推送 tag: git push origin v0.2.0
3. GitHub Actions (release.yml) 自动触发：
   ├── swift build -c release
   ├── bash script/build_and_run_codex.sh build
   └── 创建 GitHub Release + 上传 .zip
4. 在 GitHub Releases 页面补充 Release Notes
```

### CHANGELOG 格式

遵循 [Keep a Changelog](https://keepachangelog.com/)：

```markdown

## [Unreleased]

- 新增 CI/CD 自动化工作流

- 项目目录结构重组

- 修复 X 问题
```

## 7. 扩展指南

### 7.1 如何添加新数据源

1. 在 `Models.swift` 的 `TokenCostSourceFamily` 枚举中添加新 case
2. 在 `SourceProfiles.swift` 添加新 profile（默认路径、文件模式等）
3. 实现新的 Client（类似 `TokenDatabaseClient`）
4. 在 `TokenCostModel` 或新建 Store 中集成
5. 添加新的 View Page

### 7.2 如何添加新的分析卡片

1. 在 `TokenCostDashboardAnalytics` 或 `CodexDashboardAnalytics` 中添加新的结构体和计算属性
2. 在对应 PageView 中添加 `TokenSectionCard` + `TokenMetricCard`

### 7.3 如何添加新主题

1. 在 `Theme.swift` 的 `TokenCostThemeChoice` 枚举中添加 case
2. 在 `ThemePalette.swift` 的 `init(theme:)` 中添加颜色定义

## 8. 安全边界

- 本应用**只读**访问 OpenCode 数据库和 Codex session 文件
- 不修改任何源数据文件
- 配置和快照仅写入 `~/Library/Application Support/com.yanghaoran.CodexTokenCost/`
- Helper 子进程通过标准 OS 进程隔离运行
- 文件写入通过 `SafeFileStore` 防止路径穿越
- 无网络请求、无数据上传
