# v0.5.0 Release Notes

Token Cost App - OC Codex 的重大功能版。核心新增：**版本更新检查**、**余额实时监控**、**未订阅选项**。

---

## 版本更新检查 🆕

启动时每 24 小时自动检查 GitHub Release 最新版本。工具栏右侧以胶囊标签显示「更新」，点击自动下载，标签背景以左→右渐进填充可视化进度。下载完成后文件大小与 Content-Length 交叉校验，点击「安装」打开已解压的新版本 .app。全程不上传任何用户数据，仅发起匿名 GET 请求到 GitHub 公开 API。

## 余额实时监控

- **三 Provider 支持**：OpenCode Go（5h / 每周 / 每月三窗口）、Codex（credit）、OpenCode Zen（用量百分比）
- **安全设计**：API key 从本地 `auth.json` 临时读取至内存，30 秒后清除，不写磁盘。OpenCode Go authCookie 通过 macOS Keychain 加密存储
- **可视化**：仪表盘、详情页、菜单栏均展示余额梯度色条和百分比；菜单栏用紧凑进度条替代纯文字
- **默认关闭**（`balanceEnabled=false`），保持纯本地承诺

## 计费方案完善

- 四个 Provider 计费卡片新增「订阅该方案」Toggle 开关。关闭后该 Provider 不计入总成本（`monthlyUSD = nil`），灵活应对实际未订阅场景
- `BillingPlanSelection.isSubscribed` 字段向前兼容旧 JSON（缺 key 默认 true）
- 总计页总成本纳入全部四个 Provider（OpenCode + Codex + MiniMax + Xiaomi MiMo）
- 总计页 OpenCode 计价卡片 subtitle 按 API / 订阅模式动态切换

## UI/UX 改进

- **全局工具栏刷新按钮**：总计页→「刷新全部」，OpenCode 页→「重新扫描」，Codex 页→「刷新 Codex」
- **刷新进度条**：TabView 上方线性进度指示器
- **总计页卡片布局**：固定 4 列 `GridItem(.flexible())` + 统一高度对齐
- **定价文档弹窗重写**：分段解析标题/引用块/表格，原生 `HStack`+`Divider` 渲染，过滤分隔符行
- **OpenCode 详情页**：来源卡片 3 列等宽对齐；来源修改时间从 UTC 转为本地时区
- **菜单栏余额**：紧凑进度条替代纯文字显示

## 数据修复

- OpenCode 详情页来源修改时间显示本地时区（`TokenCostFormatters.localDateTime`）
- Codex 总览卡片 subtitle 移除对 Codex 数据源存在的强制依赖
- 定价文档表格 markdown 分隔符行（`|---|---|`）不再被误渲染为数据行

## 内部变更

- `BillingPlanSelection` 从编译器合成 Codable 改为手动实现（支持 `isSubscribed` 向前兼容）
- `ResolvedBillingPlan` 新增 `isSubscribed: Bool`
- `AppPreferences` 新增 `balanceEnabled`、`balanceRefreshMinutes`、`opencodeGoWorkspaceID`
- Release 目录重组：`dist/` → `release/`，新增 `release/latest/`、`release/versions.json`
- OpenCode Go 从 API 模式迁移到 Dashboard 配额模式（解析 HTML 获取多窗口额度）
- OpenCode Zen 费用去重（总费用减去 Go 模型成本）
- `BalanceSnapshot` 新增 `tertiaryWindow*` 三字段支持 Go 每月窗口
- SECURITY.md 新增 Keychain 安全声明

## 文档

- 新增 `docs/功能模块关联清单.md`：完整功能管线、数据源流程、交叉依赖矩阵、约束边界
- `docs/开发手册.md` 新增 7.4 节「版本更新检查机制」架构说明
- `docs/架构逻辑链图.md` Mermaid 图纳入 UpdateChecker
- 安全边界声明更新：余额监控 + 版本更新检查为可选网络功能
