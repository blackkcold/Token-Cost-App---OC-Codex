## 设置持久化加固 / Settings Persistence Hardening 🛡️

全局 Theme 从多存储文件迁移至 `AppPreferences` 统一管理，彻底修复 theme 不一致问题。新增 `scenePhase` 终止保存钩子，确保 App 退入后台时所有设置域最终落盘，防止系统强杀导致数据丢失。

所有设置通过 `.atomic` 原子写入，覆盖前保留最近 10 份备份。移除会误判失败并阻塞主线程的写后同步回读校验，同时修复 snake_case JSON 解码中 `ID`/`USD` 缩写字段不匹配导致的设置重启后回退到默认值。引入内存缓存 + 写穿透模式优化性能。

涉及文件：`AppPreferences.swift`, `AppPreferencesModel.swift`, `SettingsStore.swift`, `SafeFileStore.swift`, `Models.swift`, `TokenCostApp.swift`, `ContentView.swift`

## DeepSeek API 计费支持 / DeepSeek API Support 🔵

`DashboardAnalytics` 的 `apiCost` 计算从仅覆盖少数 Provider 扩展至全 Provider，`providerEffectiveCosts` 在 rawCost 为 0 时兜底 `syntheticApiCost`。新增 `deepseek-chat`/`deepseek-reasoner` → `deepseek-v4-flash`/`deepseek-v4-pro` 模型别名映射，确保 DeepSeek 模型费用正确计入总览。

内置 `Pricing.md` 新增 DeepSeek API 中英双语价目表，新增 `deepseek-api-cn-monthly` 预设方案（¥50/月估）。DeepSeek V4-Pro 促销价到期后正价已更新：input $1.74、output $3.48、cacheRead $0.0145。

涉及文件：`DashboardAnalytics.swift`, `TotalView.swift`, `BillingPlanCatalog.swift`, `Pricing.md`, `docs/Provider 计费定价速查.md`

## 统一计费计算模型 / Unified Billing Model 🔄

彻底重构计费体系，是 v0.6.0 最重要的架构升级：

- `billingOverridesByProviderKey()` 覆盖全部 5 个 Provider（OpenCode Go / Zen / ChatGPT / MiniMax / Xiaomi MiMo），不再仅限 OpenCode
- 每个 Provider 独立判断：已启用固定订阅用月费，否则走 API 估算
- 新增 `openCodeOverviewCost()` 和 `combinedMonthlyCost()` 统一入口，`TotalView` 和 `MenuBarView` 共用同一公式
- 降级 `DashboardAnalytics` 中硬编码订阅表为 `legacyFallbackMonthlyCosts`，用户方案优先
- 总成本 UI 文案明确为「已启用订阅费用 + 未订阅部分 API 估算成本」
- 全部订阅关闭时总成本全部按 API 估算

**消除的历史问题**：总计页总成本双重计数、取消订阅后费用反涨、OpenCode 硬编码 fallback $10、billingOverride 范围错误等。

涉及文件：`BillingPlanCatalog.swift`, `DashboardAnalytics.swift`, `TotalView.swift`, `MenuBarView.swift`, `Localizable.strings`, `Pricing.md`

## Keychain 与凭证改进 / Keychain Improvements 🔑

- **静默读取**：`SecItemCopyMatching` 使用 `kSecUseAuthenticationUISkip`，已有授权静默返回，无授权不弹窗
- **内存缓存**：`discoverCredentials()` 添加缓存避免同 session 重复 Keychain 访问
- **设备锁定**：`kSecAttrAccessible` 改为 `WhenUnlockedThisDeviceOnly`，不跨设备 iCloud 同步
- **写入控制**：自动刷新链路只读凭证，不自动写入 Keychain，启动时不再写入 workspaceID

涉及文件：`SecureCredentialStore.swift`, `AppPreferencesModel.swift`
