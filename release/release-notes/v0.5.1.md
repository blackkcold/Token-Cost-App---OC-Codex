# v0.5.1 Release Notes

Token Cost App - OC Codex 的功能增强与修复版。核心新增：**浏览器凭证自动导入**、**设置页测试连接**、**菜单栏刷新统一**、**版本更新检查交互重构**。

---

## 浏览器凭证自动导入 🆕

从 Edge / Chrome / Brave / Arc 自动提取 `opencode.ai` 的 Auth Cookie 和 Workspace ID，通过 macOS Keychain + PBKDF2 + AES-128-CBC 本地解密后存入钥匙串。支持交叉合并——当浏览器只找到 cookie 而 workspaceID 已在 Keychain（或反之）时，自动拼合完整凭证。

涉及新增模块：
- `CCryptoBridge/` — CommonCrypto 桥接层（PBKDF2 + AES 解密）
- `BrowserCookieExtractor.swift` — 多浏览器 cookie 提取器

## 设置页测试连接

OpenCode Go 设置页新增「测试连接」按钮，独立校验当前凭证配置，不触发全量余额刷新和 backoff 机制。

## 菜单栏刷新统一

菜单栏中三个独立刷新按钮合并为一个「刷新全部」按钮，快捷键简化为单一 `Cmd+R`。余额区域底部增加「刷新余额」按钮。

## 版本更新检查交互重构

- **启动自动静默检查**：保留 24h 缓存
- **手动检查**：工具栏新增「检查更新」按钮，即时调用 GitHub API 忽略缓存
- **有更新时**：显示「更新」/「稍后」双按钮，替代原直接下载
- **无更新时**：显示「已是最新版本 vX.Y.Z」，3 秒自动消失
- 新增 `manualCheck()`、`dismissUpdate()` 方法

## 数据修复

- **OpenCode Go 模型成本扣减**：增强模型名称匹配，解决 Zen 费用 Go 模型成本扣减不完全
- **WorkspaceID 持久化**：保存时同步写入 Keychain
- **Dashboard HTML 解析**：从 Next.js 格式迁移到 SolidJS SSR hydration，三轮窗口各两组正则改为容错模式
- **解析失败分层提示**：区分「格式变更」vs「cookie/workspace 不匹配或未订阅」，DEBUG 模式输出 HTML dump
- **浏览器导入交叉合并**：不再丢弃部分凭据

## UI 改进

- 模型分布与 Provider 分布饼图卡片等高（`HStack` + `PreferenceKey` 方案）

## 测试

- 新增 `CodexTokenCostCoreTests` 中 331 行浏览器 cookie 解密相关单元测试
